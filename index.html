422 Studios Front End Skeleton

This is a gulp / bower based front end skeleton pre-installed with Jquery and Bootstrap. We utilize bower to download dependencies like jquery, boostrap js and more. Check out https://bower.io/ for more information on bower and http://gulpjs.com/

Getting Started

You need to have the following things globally installed on your local machine. You will only run through this setup once, then its good to go for creating as many projects with the 422 Frontend Skeleton as you'd like on your local machine. If you have previously run this setup, skip to the next section.
  1. Install Node.js & NPM You need to have Node.js and NPM installed on your machine. Follow these instructions: https://docs.npmjs.com/getting-started/installing-node
  2. Install Grunt Once NPM is installed you need Grunt installed on your machine globally. Run "sudo npm install -global grunt" (minus sudo on windows).
  3. Install Gulp You also need the Gulp CLI installed locally on your machine. Run "sudo npm install -global gulp-cli" (minus sudo on windows)
The following applies to each time that you start a new project with this skeleton.
  1. Customize Package.json" Open "package.json" in the root directory. Update the name, version, description, keywords and author if you desire, this is not a required step.
  2. Check for outdated NPM Packages Via Terminal or Git Bash, change into your project directory. Run the command "npm outdated". This command should return nothing. If nothing returns, then you are good to go and you can begin your project.
  3. Update NPM Packages if Necessary If something is returned from the above output it means that some of the NPM packages used in the skeleton are outdated. At your own risk, run "npm update" to update all packages. This may or may not cause unexpected errors or complications which will require some "on your own" troubleshooting to address and fix.

Downloading & Installing Bower Packages

Bower packages will download and install to "components/libs". You will need to open the "gulpfile.js" file in the root folder and scroll to the section "uglify plugins" for more information on adding bower plugins to the automatic "build system" created with gulp.

The short version is that you will need to open "gulpfile.js" every time you download a bower component and add the CSS / JS to the appropriate gulp sections. It's super easy, so don't be intimidated. A quick read through of the gulp file is probably enough to understand it fully.

Compiling JS and SCSS Changes

All that you have to do to start gulp is simply use the git bash application on windows or terminal on mac os and change into your project directory. Then use the command, "gulp watch". From there, any changes in the "components/scss" and "components/js" folders will trigger a full build of all bower components and custom js / scss files. They get compiled to 4 js / css files in the "dist" folder. The index.html template is already set up to include the appropriate files.

SCSS Instructions The grunt build system for this template looks to the "components/scss/style.scss" file for its target. You may add imports to this file if you want. We have set up a "libs" folder for things like the "animate.css" library or other css libraries not downloaded via bower. Look for bower libraries first before installing your own here because the bower libraries are easier to keep up with and who doesn't want to be a hipster and use the command line :D We have also included a "structure" folder to put app-wide styles and a "partials" folder for individual layouts. This is how we do it at 422 Studios, but you can change the folders and import setup inside the "components/scss" folder if you want to do it your own way, just make sure to make the appropriate changes to the gulp target, "components/scss/style.scss". As a bonus, we've even included some of the standard scss files that we use for variables, wrapper set up and more. Feel free to delete and start over if you feel it necessary.

Images & Fonts

To follow the convention of the template, store your images and font files in the "dist/images" / "dist/fonts" folders.
